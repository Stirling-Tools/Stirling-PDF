# APT Repository — Setup & Usage Guide

## For users: installing Stirling-PDF via APT

### 1. Add the repository

```bash
# Download and trust the Stirling-PDF GPG key
curl -fsSL https://stirling-tools.github.io/Stirling-PDF/KEY.gpg \
  | sudo gpg --dearmor -o /usr/share/keyrings/stirling-pdf.gpg

# Add the sources.list entry
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/stirling-pdf.gpg] \
  https://stirling-tools.github.io/Stirling-PDF stable main" \
  | sudo tee /etc/apt/sources.list.d/stirling-pdf.list
```

### 2. Install

```bash
sudo apt update
sudo apt install stirling-pdf
```

### 3. Run

```bash
# Start as a systemd service (if packaged with service file)
sudo systemctl enable --now stirling-pdf
# Then open http://localhost:8080
```

## For the maintainer: one-time setup

### Step 1 — Generate a GPG signing key

Run this on a secure machine (or use an existing org key):

```bash
gpg --full-generate-key
# Choose: RSA, 4096 bits, no expiry, name: "Stirling-PDF Signing", email: contact@stirlingpdf.com
```

Export the key ID:

```bash
gpg --list-secret-keys --keyid-format LONG
# Note the key ID after "rsa4096/"
```

Export private key for GitHub Actions:

```bash
gpg --armor --export-secret-key <KEY_ID> > stirling-pdf-signing.asc
```

Export public key (committed to gh-pages-apt as KEY.gpg automatically by CI):

```bash
gpg --armor --export <KEY_ID> > KEY.gpg
```

### Step 2 — Add GitHub Secrets

Go to **Settings → Secrets and variables → Actions** in the repository and add:

| Secret name | Value |
|-------------|-------|
| `APT_GPG_PRIVATE_KEY` | Contents of `stirling-pdf-signing.asc` |
| `APT_GPG_PASSPHRASE` | The passphrase you chose when generating the key |

### Step 3 — Enable GitHub Pages for `gh-pages-apt` branch

1. Go to **Settings → Pages**.
2. Source: **Deploy from a branch**.
3. Branch: `gh-pages-apt`, folder: `/` (root).
4. Save.

The workflow also uses the **Actions → Deploy to GitHub Pages** approach — make
sure the `pages` permission is allowed under **Settings → Actions → General →
Workflow permissions**.

### Step 4 — First run

Publish a release (or trigger the workflow manually via **Actions →
Publish APT Repository → Run workflow**) with a valid release tag.

## Local Testing

Validate the repository and package locally before pushing to GitHub Pages.

### Prerequisites

```bash
sudo apt-get install -y dpkg-dev gnupg apt-utils
```

### Step 1 — Build the local repo

```bash
# Create a scratch directory
mkdir -p /tmp/stirling-apt-test
cp /path/to/stirling-pdf_<version>_amd64.deb /tmp/stirling-apt-test/

# Generate the Packages index
cd /tmp/stirling-apt-test
dpkg-scanpackages --arch amd64 . > Packages
gzip -k Packages

# Verify the index lists the package
grep -E "^(Package|Version|Filename)" Packages
```

### Step 2 — Add as a local apt source

```bash
# Point apt at the local directory
echo "deb [trusted=yes] file:///tmp/stirling-apt-test ./" \
  | sudo tee /etc/apt/sources.list.d/stirling-pdf-local.list

sudo apt-get update
```

> **Gotcha:** `file://` URIs require the path to contain a flat `Packages` file
> alongside the `.deb`s. The `dpkg-scanpackages .` above produces exactly that.

### Step 3 — Install from the local repo

```bash
sudo apt-get install stirling-pdf

# Confirm the package is installed and check version
dpkg -l stirling-pdf
dpkg -s stirling-pdf | grep -E "^(Status|Version|Depends)"
```

### Step 4 — Test GPG signing

```bash
# Sign the Packages file with your key (same key used in CI)
export GPG_KEY_ID=<your-key-id>
gpg --armor --detach-sign --default-key "$GPG_KEY_ID" \
    --output Packages.gpg Packages

# Verify the signature
gpg --verify Packages.gpg Packages && echo "Signature OK"

# List keys apt trusts (for installed keyrings)
apt-key list 2>/dev/null || gpg --no-default-keyring \
  --keyring /usr/share/keyrings/stirling-pdf.gpg --list-keys
```

> **Gotcha:** Modern apt uses `signed-by=` in sources.list rather than the
> legacy `apt-key` trust store. Use the `gpg --verify` check above; `apt-key
> list` is deprecated but still useful for spot-checking key presence.

### Step 5 — Verify package contents

```bash
# List everything the .deb installs
dpkg -L stirling-pdf

# Check the postinst / control scripts
dpkg-deb --info /tmp/stirling-apt-test/stirling-pdf_<version>_amd64.deb
dpkg-deb --contents /tmp/stirling-apt-test/stirling-pdf_<version>_amd64.deb
```

### Cleanup

```bash
sudo apt-get remove --purge stirling-pdf
sudo rm /etc/apt/sources.list.d/stirling-pdf-local.list
sudo apt-get update
rm -rf /tmp/stirling-apt-test
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `NO_PUBKEY` error on `apt update` | Re-run the `curl` key-import command |
| `Release file is not valid yet` | Clock skew — check system time |
| `file://` source returns 404 | Ensure `Packages` (uncompressed) exists in the same directory as the `.deb` |
| `dpkg-scanpackages` not found | `sudo apt-get install dpkg-dev` |
| Workflow fails at signing step | Verify `APT_GPG_PRIVATE_KEY` / `APT_GPG_PASSPHRASE` secrets are set correctly |
